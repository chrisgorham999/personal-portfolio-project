API Unit Tests

WEB 420 API Unit Test 1 - findComposerById (Swagger)
Demonstrates the findComposerById API functionality in Swagger. A user enters the ID number of a composer and returns a result that matches the ID number.
WEB 420 API Unit Test 2 - findAllComposers (SoapUI)
Demonstrates the findAllComposers API functionality in SoapUI. This request displays all documents in the Composer API.
WEB 420 API Unit Test 3 - createComposer (SoapUI)
Demonstrates the createComposer API functionality in SoapUI. This request adds a Composer Document to the Composer API.
WEB 420 API Unit Test 4 - createPerson(Swagger)
Demonstrates the createPerson API functionality in Swagger. This request creates a new Person document in the database.
WEB 420 API Unit Test 5 - findAllPersons(SoapUI)
Demonstrates the findAllPersons API functionality in Soap UI. This request displays all Person documents in the database.
WEB 420 API Unit Test 6 - signup (Swagger)
Demonstrates the Signup API functionality in Swagger. This request creates a new user name, password, and email address.
WEB 420 API Unit Test 7 - login (SoapUI)
Demonstrates the Login API functionality in SoapUI. This request logs in using a previously created user name and password.
WEB 420 API Unit Test 8 - createCustomer (SoapUI)
Demonstrates the create customer API functionality in SoapUI. This request creates a new customer document with a first name, last name, and user name.
WEB 420 API Unit Test 9 - findAllInvoicesByUserName (SoapUI)
Demonstrates the find invoices by user name API functionality in SoapUI. This request displays all invoices from a user name that is submitted by the user.
WEB 420 API Unit Test 10 - createInvoiceByUserName (Swagger)
Demonstrates the create invoice by user name API functionality in Swagger. This request will create an invoice for a specific user name that is entered by the user.
WEB 420 API Unit Test 11 - updateComposerById (Swagger)
Demonstrates the update composer document by ID API functionality in Swagger. This request will update a composer document for a specific ID that is entered by the user.
WEB 420 API Unit Test 12 - deleteComposerById (SoapUI)
Demonstrates the delete composer document by ID API functionality in SoapUI. This request will delete a composer document for a specific ID that is entered by the user.
WEB 420 API Unit Test 13 - Capstone Project
Demonstrates four separate operations: findAllTeams(displays a list of all team documents), assignPlayerToTeam (adds a player to the players array for a team), findAllPlayersByTeamId (lists all players in the players array for a specific team ID that is inputted by the user), and deleteTeamById (deletes a team document by the user entering a team ID)
